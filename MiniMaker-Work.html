Interface de création et d'édition pour les procédures au standard iXi

Bienvenue dans iXiMAKER Community - WORK

Cet éditeur vous permet de créer et de modifier vos procédures XR.
Utilisez les onglets ci-dessus pour naviguer entre les différentes sections de l'éditeur.

  • automation Procédure : Définissez les paramètres généraux et spécifiques de votre procédure.
  • step_over Étapes : Ajoutez, modifiez et organisez les étapes de votre procédure et leurs actions
  • timeline Stages : Gérez les stages regroupant plusieurs étapes.
  • swap_vert Import/Export : Importez des fichiers JSON existants ou exportez votre procédure.

task_alt Préparation

Avant de commencer la création de votre procédure, vérifiez les points suivants :

counter_1 Création des dossiers

  • Créez votre structure de dossiers comme indiqué, ci-dessous, dans la section "Aide"

counter_2 Préparation des médias

  • Préparez vos médias et copiez-les dans le dossier "XXXX_FR"
    • Images : format jpg, png ou gif (Taille conseillée : 2048px)
    • Audios : format ogg ou mp3
    • Modèles 3D : format glb ou gltf avec textures incluses (10Mo max - 4Mo conseillé)

    ⚠️ Les noms de fichiers et de dossier ne doivent pas contenir de caractères spéciaux, ni accentués tels que : (&é~#à%*$ê/!_\)

info Pour les modifications

Si vous souhaitez éditer un fichier existant :

  • Cliquez sur l'onglet "Import/Export"
  • Importez votre json WORK via le bouton "Importer" (pas de LEARN)
  • Suivez les indications de la section "Pour commencer"

⭐ Pour commencer

  • Cliquez sur l'onglet "Procédure" pour définir les paramètres de base.
  • Cliquez sur l'onglet "Étapes" pour créer vos étapes et vos actions
  • Cliquez sur l'onglet "Stages" pour organiser vos étapes
  • Cliquez sur l'onglet "Import/Export" pour sauvegarder votre json
    (⚠️le nom doit être exactement le même que votre dossier, avec l'extension .json)

👉N'oubliez pas de sauvegarder régulièrement votre travail

  • Utilisez le bouton en bas de page : ▶ Générer JSON
  • Rendez-vous dans l'onglet "Import/Export"
  • Cliquez sur 📤 Exporter puis l''onglet "Import/Export".

🛟 Aide

Fonctionnement

iXiMAKER Community est un éditeur visuel pour vous assister dans l'édition et la création de fichiers JSON iXi.
🆗 Il permet :

  • de simplifier la création de procédures XR
  • d'y assiocer les liens vers les fichiers média
  • de créer des fichiers JSON conformes au format iXi
  • d'importer et de modifier des fichiers JSON WORK conformes au format iXi

🚫 Il ne permet pas :

  • de copier un fichier media dans le dossier (uniquement version pro)
  • de spatialiser vos contenus (c'est le rôle de "iXiWORK - Place" 😉)
  • de créer un dossier de contenu (uniquement version pro)

Structure des fichiers au standard iXi

Pour fonctionner, les dossiers et fichiers doivent respecter l'organisation suivante :

  • 📁Machine XYZ (Votre "Référentiel" contenant vos procédures et vos formations dans les différentes langues)
    • 📁Machine XYZ_FR (Variante linguistique définie selon le champ "Code pays")
      • 📄Machine XYZ_FR.json (Le JSON de la procédure)
      • 🧊mainHologram.bytes (le modèle 3D principal au format .glb ou .gltf, renommé en .bytes)
      • 🧊les_autres_modeles.bytes (les autres modèles 3D au format .glb ou .gltf, renommé en .bytes)
      • ...
      • 🖼️les_images.png
      • ...
      • 🔊les_audio.ogg
      • ...
      • 📁learn (contenant les fichiers images, sons, modèles 3D référencés dans la formation)
        • Voir l'aide d'iXiMAKER LEARN pour plus d'informations sur les fichiers de ce contenu.

    • Nommage des fichiers

    Pour les noms de fichiers, et afin d'être pleinement compatible avec la version PRO d'iXiMAKER, il est conseiller d'utiliser la syntaxe suivante :

    • 🧊 Hologrammes :
      • mainHologram.bytes : pour le modèle 3d principal
      • step-1_hologram.bytes : pour mettre à jour l'hologramme à partir de l'étape 1
      • step-3_action-1_hologram.bytes : pour un hologramme spécifique pour l'action 1 de l'étape 3
    • 🖼️ Images :
      • step-1_img.png : pour l'illustration de l'étape 1
      • step-2_action-1_img.png : pour l'illustration de la 1ere action de l'étape 2
    • 🔊 Audio :
      • step-1_audio.ogg : pour l'audio de l'étape 1
      • step-2_action-1_audio.ogg : pour l'audio de la 1ere action de l'étape 2
